# Architecture

## Five-role operating model

```text
AXIS OMNI
├── AXIS — implementation
├── VERITY — validation
├── DIRECTOR — creative
└── BEACON — intelligence
```

## Detailed specialist modules

The five roles may activate the fourteen modules in `prompts/modules` when separation improves reliability, auditability, or scale.

## Core state objects

- Supplier
- Product
- Authorization
- Listing
- Experiment
- Order
- Financial waterfall
- Supplier performance
- Agent handoff
- Approval event
- Audit event

## Execution boundary

AXIS OMNI is not allowed to claim live deployment without authenticated tools, authorized accounts, and a verified platform response. Prompt output alone is not execution.
